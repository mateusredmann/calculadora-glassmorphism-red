## Story
**As a** usuário,
**I want** que a calculadora trate entradas inválidas graciosamente,
**so that** erros não quebrem minha experiência de uso.

## Acceptance Criteria
1. Divisão por zero exibe `"Erro"` no display e reseta o estado ao clicar em AC
2. Ponto decimal (.) não pode ser inserido duas vezes no mesmo número
3. Operador não pode ser inserido se não há valor atual válido
4. Resultado muito longo é formatado com `toFixed()` ou notação exponencial para caber no display
5. Clicar em `=` sem operador definido não altera o valor atual
6. Após exibir resultado, digitar novo número inicia nova entrada (não concatena ao resultado)

## Dev Notes

**Prerequisite:** Stories 1.1 e 1.2 concluídas — lógica core implementada em `script.js`.

**Guard de estado de erro — padrão recomendado:**
```javascript
function handleButtonClick(e) {
  const btn = e.target.closest('button');
  if (!btn) return;
  
  // Guard: se estado é erro, apenas AC funciona
  if (state.currentValue === 'Erro' && btn.dataset.action !== 'clear') return;
  
  // ... resto do handler
}
```

**Prevenção de ponto duplo:**
```javascript
function handleDecimal() {
  if (state.currentValue.includes('.')) return; // guard
  state.currentValue += '.';
  updateDisplay();
}
```

**Formatação de resultado longo:**
```javascript
function formatResult(value) {
  if (value === 'Erro') return value;
  const num = parseFloat(value);
  if (isNaN(num)) return 'Erro';
  const str = String(num);
  if (str.length <= 10) return str;
  return num.toPrecision(8).replace(/\.?0+$/, '');
}
```

**Referência PRD:** FR8 (divisão zero), FR2 (decimal), FR12 (encadeamento) — `docs/prd.md`

### Testing
- Divisão por zero: `5 ÷ 0 =` → display mostra `"Erro"`, AC restaura
- Decimal duplo: `1 . 5 .` → display mostra `1.5` (segundo `.` ignorado)
- Operador sem valor: abrir app e pressionar `+` → nenhuma mudança
- Resultado longo: `1 ÷ 3 =` → formatar resultado (ex: `0.3333333`)
- Igual sem operador: `5 =` → display permanece `5`
- Nova entrada após resultado: `5 + 3 = → digitar 7` → display mostra `7` (não `87`)
