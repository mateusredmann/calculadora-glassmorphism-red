## Story
**As a** usuário,
**I want** ver orbs coloridos animados no fundo,
**so that** a interface tenha profundidade e movimento que reforcem a identidade visual.

## Acceptance Criteria
1. Mínimo 3 orbs criados como elementos `<div class="orb">` no HTML
2. Cada orb possui: `border-radius: 50%`, `filter: blur(60px)`, `position: absolute`, opacidade entre 0.3-0.6
3. Orbs usam cores variadas da paleta vermelha: `#FF0000`, `#8B0000`, `#CC0000`
4. Cada orb possui animação CSS `@keyframes` única com `transform: translate()` e `scale()` em loop infinito
5. Duração das animações entre 15s e 25s com `ease-in-out` para movimento orgânico
6. Orbs não interferem nos cliques dos botões (`pointer-events: none`)
7. `.calculator` possui `z-index` superior aos orbs garantindo legibilidade

## Dev Notes

**Prerequisite:** Story 2.1 concluída — Custom Properties e estrutura glass existem.

**Estrutura HTML recomendada:**
```html
<body>
  <div class="calculator-wrapper">
    <div class="orb orb-1"></div>
    <div class="orb orb-2"></div>
    <div class="orb orb-3"></div>
    <div class="calculator">
      <!-- display + buttons -->
    </div>
  </div>
</body>
```

**CSS do wrapper:**
```css
.calculator-wrapper {
  position: relative;
  display: flex;
  align-items: center;
  justify-content: center;
}
```

**Por que `transform` e não `top`/`left` nas animações:**
- `transform` é processado pela GPU (compositor thread)
- Não causa layout reflow — animação a 60fps garantida
- `top`/`left` causam recálculo de layout em cada frame → jank

**Referência PRD:** FR9, NFR2 — `docs/prd.md`
